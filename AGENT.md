# AGENT - Orquestador de Generación de Código

Este agente está diseñado para orquestar la generación de código en el proyecto.

## Contexto del Proyecto

**IMPORTANTE:** Todo el contexto del proyecto, incluyendo arquitectura, flujos, decisiones de diseño y especificaciones técnicas, se encuentra documentado en la carpeta `context/`.

Antes de realizar cualquier cambio o generación de código, el agente debe:

1. **Revisar la carpeta `context/`** para entender:
   - La arquitectura actual del sistema
   - Los flujos de datos y procesos
   - Las decisiones de diseño tomadas
   - Las especificaciones técnicas
   - Los patrones y convenciones del proyecto

2. **Consultar los archivos de contexto relevantes** según la tarea a realizar

3. **Mantener consistencia** con las decisiones y patrones documentados

4. **Actualizar la documentación** en `context/` si se realizan cambios significativos en la arquitectura o flujos

## Estructura de Contexto

La carpeta `context/` contiene archivos markdown que documentan diferentes aspectos del proyecto:

- `flujo-creacion-documentos.md`: Documenta el flujo actual y propuesto para la creación de documentos, incluyendo el manejo de archivos (imágenes y texto)
- `guia-frontend-envio-archivos.md`: Guía completa para el equipo de frontend sobre cómo enviar archivos al backend usando FormData

## Instrucciones para el Agente

Cuando se solicite generar o modificar código:

1. **Leer primero** los archivos relevantes en `context/`
2. **Entender** el contexto y las decisiones de diseño
3. **Generar código** que sea consistente con la arquitectura documentada
4. **Seguir** los patrones y convenciones establecidos
5. **Mantener** la coherencia con el resto del código existente

---

**Nota:** Este archivo será actualizado con más instrucciones específicas según se vayan definiendo más reglas y patrones para la generación de código.
